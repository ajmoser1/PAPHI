Describe your process. At a high level, how did you go about creating this project?

At a high level, I planned out the project without AI. I wanted to keep it simple and easy but above all, useful. I made sure to take my time with each prompt to keep any agents or plans as closely aligned with my vision as possible.
Because it was a relatively simple site, I mostly used AI tools to explain what I didn't really understand about some of the backend, like conversations and messaging between users.


What AI tools and strategies did you use? What models did you choose, and were you using them in a browser chat, in something like Cursor, in the command line, etc? Did you use an agentic process like in HW8, or something else?

I used cursor and github copilot. Cursor was mainly used to create the foundations of the site for me to heavily modify after, and github was to help me understand the backend and properly implement it. I used cursor to help me understand a lot of the code as well so I could fine-tune the ui and layout how I wanted it.
I didn't use an agentic approach with a code reviewer but I did analyze all the errors and warnings from supabase's advisor feature. I sent those to github copilot to fix any issues with backend.


Why did you make the choices above?

I'm a big fan of cursor because for the most part it's much easier to access the code and make small changes that don't require a prompt. It's useful for people who know what they're doing for the most part and want to have more control over the code and development.
I also like being able to run a dev window within cursor and not have to switch over to the browser back and forth.


What changed from your pre-113 approach? Compare your approach to how you might have approached the project before 15-113. What has changed, if anything? Do you feel more or less well-equipped to create projects like this within a reasonable time frame?

Before 113, I definitely would've had to just send prompts over and over to get something kind of close to what I wanted. I also would've probably used chatgpt to try to send me code snippets but that would really suck as it can't send an entire codebase with different folders and everything. 
I also would've been less likely to try to read and understand the code because I would just want to see it finished.


What might you do with this project with more time?

I'm considering expanding to Pitt and having their SAE chapter sign up. I could see this going to Nationals eventually since they have something called career connect, which is essentially just a job board.
Instead, this is for attaining referrals and networking with alumni you might otherwise never meet. I don't really intend to add any more features for the time being since it isn't supposed to be a linkedin replacement or anything.
